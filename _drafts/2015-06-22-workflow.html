I know that a designers workflow can be as personal as your view on unsolicited redesigns, or the neverending discussions on whether designers should be able to code (they should, btw).

I don't think you should follow a top designers workflow (or mine) just because you admire the work they produce, or because of some passive attraction to their status in the community. Each persons workflow is unique to the work they are required, or want, to produce.

Most of my work involves new products, so for now I'll leave out user feedback, and data analysis, but know that these are important if you're working with an existing site, app or product.

My workflow is generally the following:

  • Business objectives
  • Red routes
  • Personas
  • User flows
  • Information Architecture
  • Wireframes
  • Prototyping
  • UI Design
  • Front-end coding
  • Speed enhancements

Wireframes

For web and app UI, in my opinion, wireframes are the best UX document you can provide your client with. You can cut down on a lot of wasted time in the UI Design stage by making sure the layout is locked down. It provides the client with an opportunity to make sure the content is all there and with a heirarchy.

Prototyping

Prototyping interactions has been a revelation for client approvals in the recent past. It makes it so much easier to show the client what will occur rather than leaving it to some copy and their imagination

In the last year or so, there have been a number of prototyping tools come to market. I've tried a few and so far landed on Invision for static prototyping and FramerJS for more complex problems involving animations.

UI Design

For UI I use Sketch predominantly. Not only is it lightweight and built for UI, but there is an awesome community built around it, much more so than any I've seen built around Adobe products in the 16 years I've been using them.

Front-end coding

My front-end coding mainly involves HTML and CSS, with a little Javascript thrown in. I have some favourite tools I use, plus some I'm trying to learn. I used to use Coda and Codekit for my HTML and CSS, but lately have been using Atom as it compiles LESS and SASS without the need for a separate app. I have a certain love for LESS over SASS as it seems for designer-friendly to me, but picking up SASS seems to be easier using LESS as an easier introduction to understanding the basic structures.
